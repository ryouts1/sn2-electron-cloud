# Design Decisions

## 1. 反応は 1 つに絞る
複数反応に一般化すると UI は豪華になるが、就活ポートフォリオとしては論点がぼやけやすい。  
今回は **SN2 の 1 反応に集中** して、電子の流れ・遷移状態・脱離までを丁寧に見せるほうを選んだ。

## 2. 「厳密計算」ではなく「教育用モデル」と明記する
電子雲構造という言葉だけを前面に出すと、量子化学計算をしているように誤解されやすい。  
そのため README と UI の両方で、これは **教育用の可視化モデル** であると明記している。

## 3. Canvas ではなく SVG
今回は以下の理由で SVG を採用した。

- 結合、矢印、電子雲、電荷記号をレイヤーとして扱いやすい
- DOM ベースなので調整がしやすい
- 小規模プロジェクトとして可読性を保ちやすい

## 4. ライブラリ依存を増やさない
フレームワークや描画ライブラリを入れず、ブラウザ標準 API だけで構成している。  
この方針にすると、採用側に対して「何を自分で組んだか」を見せやすい。

## 5. モデル出力を UI から独立させる
`getReactionState(progress)` が返すオブジェクトを変えれば、将来的に次の拡張が可能。

- E2 反応や付加反応への差し替え
- 3D 表示への拡張
- CSV / JSON による反応定義の外出し
- パラメータ比較モードの追加
